## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Use YAML frontmatter for .md files | Standard convention for static site generators, easy to parse |
| Use TECHNIQUE_INFO dict for .py files | Simple module-level metadata, no magic |
| Async analyze() method | Required for Claude CLI integration (subprocess) in Phase 3.3 |
| PromptStrategy raises NotImplementedError | Claude integration deferred to Phase 3.3 |
| Case-insensitive strategy names | User-friendly, matches exchange pattern |
| _strategies_loaded flag | Prevents reloading when cache is empty (fixed test issue) |

## Implementation Details

### Exception Hierarchy:
```
StrategyError (base)
├── StrategyValidationError (with field attribute)
├── StrategyExecutionError (with strategy_name attribute)
└── StrategyLoadError (with file_path attribute)
```

### TechniqueInfo Model:
- `name`: Unique identifier
- `version`: Semantic version (validated pattern)
- `description`: Brief description
- `technique_type`: "prompt" or "code"
- `symbols`: List of supported trading pairs
- `timeframes`: Recommended timeframes
- `status`: "experimental", "active", or "deprecated"
- `created_at`, `updated_at`, `changelog`: Versioning metadata

### BaseStrategy Interface:
- `__init__(info: TechniqueInfo)`
- Properties: `name`, `version`, `info`
- Abstract: `async analyze(ohlcv, symbol, timeframe) -> AnalysisResult`
- Helper: `validate_input(ohlcv, min_candles=20)`

### Factory Pattern:
- `@register_strategy("name")` decorator
- `get_strategy(name)` - Get by name
- `get_available_strategies()` - List all
- `get_strategies_by_symbol(symbol)` - Filter by symbol
- `get_strategies_by_status(status)` - Filter by status
- `clear_strategy_cache()` - Clear cache (for testing)
