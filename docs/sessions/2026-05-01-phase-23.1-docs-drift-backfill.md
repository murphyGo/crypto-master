## Key Decisions

| Decision | Rationale |
|---|---|
| Backfilled session logs carry an explicit "Backfill notice" prologue | The audit value of session logs comes from real-time fidelity to the dev's report. A backfilled log written from a commit body is a different category of artefact — useful for traceability, but not a preserved record of "what the dev wrote at the time". The prologue makes the distinction immediately visible to a future auditor so backfilled logs aren't mistaken for real-time ones. |
| Code Review Results / Verification sections in backfilled logs explicitly tagged "reconstructed" | The original commits did not preserve qa-reviewer / quant-trader-expert verdicts or test-run output. Inventing reviewer marks would defeat the audit-trail value. The honest shape is "no preserved review report; reconstruction limited to what the commit body and diff carry". |
| TECH-DEBT items section in backfilled logs says "None new" with forward-references | The commit diffs for `094a79d` and `ab9dc32` did not surface any TECH-DEBT items at the time. Subsequent-cycle items (DEBT-027, DEBT-028) that are *related* to the persistence-path patterns those commits touched are noted in the Potential Risks section as "later addressed by Phase 22.1 / 22.2" forward-references rather than retrofitted into the TECH-DEBT items section as if the original cycle had surfaced them. |
| ADR-list verification in DESIGN.md skipped (not needed) | The Phase 23.1 spec said "verify the ADR list also matches". Investigation confirmed DESIGN.md does not currently carry an ADR list, and the project has no `docs/adr/` directory in the current checkout. No corresponding ADR-list edit was needed; documented in this session log so a future audit doesn't relitigate the spec item. |
| `CLAUDE.md` tree lists representative files per directory rather than just naming the directory | The original tree only named directories (e.g. `exchange/`). The drift fix could have just added bare directory names (`runtime/`, `tools/`, `utils/`) but adding representative file lists (e.g. `runtime/` → "engine.py, activity_log.py, jsonl_rotator.py") makes the tree useful for orientation: a new contributor reading `CLAUDE.md` can immediately see "the engine lives in runtime/" without grepping. The cost is mild duplication with the actual filesystem, but the tree is small enough that drift is recoverable. |
| Phase 23.1 status table flipped without sealing Phase 23 | Phase 23 has two sub-tasks (23.1 + 23.2). 23.1 is the audit-finding cleanup; 23.2 is the Phase 17.2 / 17.3 numbering reconciliation. Sealing Phase 23 requires both. 23.1 status flips to ✅ Complete; the phase-level row continues to read whatever it read pre-cycle (no phase-seal cross-check this cycle). |

## Potential Risks

- **Backfilled session logs are not authoritative for what
  was reviewed at the time.** Future audits relying on session
  logs to reconstruct a cycle's review verdict for Phase 17.2
  / 17.3 will find "(Reconstructed)" markers in the Code
  Review Results section — accurate but not as informative as
  a real-time log would have been. Mitigation: the backfill-
  notice prologue makes the limitation explicit; the commit
  body and diff are still the canonical source for what
  shipped.

- **TECH-DEBT.md statistics depend on `### DEBT-` heading
  count discipline.** The recomputation this cycle was correct
  for the current state, but the count is a one-pass snapshot
  — a future cycle that adds or resolves a DEBT must update
  the Statistics table by hand. The original DEBT-037
  description named this as a sub-issue ("Statistics row would
  need to be re-counted after each new debt landing"); this
  cycle resolves the snapshot but not the maintenance burden.
  Folding the Statistics computation into a CI check or
  pre-commit hook would close the maintenance gap; out of
  scope for 23.1.

- **DESIGN.md ADR-list verification is contingent on the
  project's ADR practice.** This cycle confirmed no ADR list
  exists in DESIGN.md and no `docs/adr/` directory is present.
  If a future ADR landing pattern reintroduces a DESIGN.md ADR
  list section, the cross-reference check will need to be
  rerun.
