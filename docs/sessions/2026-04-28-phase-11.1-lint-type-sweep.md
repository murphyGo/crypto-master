## Work Summary

Phase 11.1 resolves DEBT-001 (Medium): the 18 pre-existing ruff
errors + 24 pre-existing mypy errors that Phase 10.5's
touch-and-verify discipline first surfaced. The cycle is pure
typing/lint hygiene — no behaviour change, no new tests, zero
`# noqa` / `# type: ignore` added. 1083 tests pass before and after.

In-scope numbers: 18 ruff → 0; 12 in-scope mypy → 0; total mypy
39 → 29 (-10). The remaining 29 mypy errors are the explicitly
out-of-scope clusters: ccxt typing in `src/exchange/binance.py`
(blocked on missing stubs), `src/exchange/factory.py` shape drift
(needs quant-trader review before fixing), Streamlit type errors
across four dashboard files, and a single lambda annotation in
`src/main.py:220`. Each of these is recorded as its own DEBT item
(DEBT-005 through DEBT-008) so the post-11.1 baseline-clean
contract is "ruff + in-scope mypy clean", with the four module
clusters tracked explicitly rather than left as ambient noise.

The non-obvious choice was the depth of the fixes. Two approaches
were considered: scoped `# type: ignore` per call site (cheap,
quick, but leaves the hygiene gap permanently), versus actually
typing the call sites correctly (more work but produces a usable
gate signal afterwards). Dev took the latter — `Order` was
imported into `src/trading/live.py` and `src/trading/paper.py` so
return-type widening could be expressed correctly, `closing_side`
was retyped `Literal["buy", "sell"]` to satisfy the narrowing
mypy needs at the order call site, `fm.get(...) or fallback` in
`src/ai/improver.py` was coerced to `str` at parse-time so
downstream type flow stays clean, and `src/backtest/analyzer.py`
got an explicit `float(...)` cast for the no-any-return path.
B904 raise-from was applied uniformly with `from e`. UP035 was
fixed by importing `Callable` from `collections.abc` in
`src/strategy/factory.py`. Test files (F401 / F841 / I001) were
cleaned via `ruff --fix`. Tooling: `pyproject.toml` ruff config
moved from deprecated top-level `select` / `ignore` to
`[tool.ruff.lint]`, dropping a deprecation warning on ruff 0.15.9.
`types-PyYAML>=6.0` added to dev extras.

The cycle also shipped a small `scripts/lint.sh` so future cycles
can gate on the clean baseline. QA flagged a 🟡 on this script:
line 6 uses `ruff check src tests --fix`, which silently rewrites
source on lintable regressions instead of reporting them. Fine for
local dev convenience; unsafe for a CI gate. Recorded as
DEBT-009. The script ships as-is for now since the dev convenience
is genuine; the future split into `lint.sh` (CI report-only) +
`lint-fix.sh` (dev with `--fix`) is captured as the resolution.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Type the call sites rather than `# type: ignore` per error | A `# type: ignore` sweep would clear the error count without producing a useful gate signal. Typing the calls correctly leaves a clean baseline future cycles can rely on, and surfaces real type-narrowing on the live trading path (`Order` returns, `Literal["buy","sell"]` for closing_side). Zero suppressions added across the cycle. |
| Out-of-scope clusters split into per-module DEBT items | The 4 module clusters (binance ccxt, factory shape drift, dashboard Streamlit, main lambda) each need different treatment: stubs / Protocol, expert review, mini-sweep, one-line fix. Bundling them under one umbrella DEBT-XXX would obscure that they have different fix shapes and different urgency. Split into DEBT-005 / 006 / 007 / 008 so each tracks its own resolution path. |
| `scripts/lint.sh` shipped with `--fix` despite qa 🟡 | Dev convenience is the primary use case today; the script is operator-invoked, not in CI. QA's 🟡 is the warning that **if** wired to CI it has to be split first. Recorded as DEBT-009 with the explicit "drop `--fix` or split into lint.sh + lint-fix.sh" resolution. Not worth blocking the cycle on. |
| Ruff config migration to `[tool.ruff.lint]` | Ruff 0.15.9 emits a deprecation warning for top-level `select` / `ignore` keys. The migration is bit-for-bit equivalent (qa verified) and drops a recurring warning. Picked up in passing during 11.1 since the cycle was already touching ruff config. |
| `types-PyYAML` to dev extras, not main deps | YAML parsing is dev/tooling/test territory; main deps don't need the stub package. Same convention as other typing-stubs additions in the project. |
