## Potential Risks

- **Renumber-notice prologue is a session-log convention not
  enforced anywhere in tooling.** A future audit that machine-greps
  for `# Session Log: YYYY-MM-DD - Phase N.M` to extract phase
  numbers will get the post-rename header for the 17.4 log
  correctly, but the in-prose references to "Phase 17.2" inside
  the body would mis-attribute work to 17.2 if the grep walked the
  full body. Mitigation: the renumber-notice prologue is the first
  block-quote in the file, before any prose; an audit tool that
  reads only the title + the first block-quote will get the
  disambiguation.

- **Change-history erratum parentheticals add visual noise to the
  table.** Three rows now carry an inline parenthetical referencing
  Phase 23.2; a fourth row (DEBT-020 erratum) carries a similar
  parenthetical. The trade-off is between (a) visual cleanliness
  and (b) traceability for a reader grep'ing for "17.2". This
  cycle picked (b); future cycles touching these rows should not
  remove the parentheticals without reading this session log.

- **The Phase 17 Requirements-Mapping row expansion (FR-031 /
  NFR-008 / FR-005 / FR-021 added) is correct but goes beyond the
  literal "verify" wording in the 23.2 spec.** A strict reading
  would have flagged the Requirements-Mapping row gap back to the
  planner for an explicit edit cycle. The pragmatic choice this
  cycle was to absorb the four-FR addition into the row directly
  because (i) the FRs were already cited in the 23.1-backfilled
  spec bodies, (ii) the verify step's purpose is exactly to catch
  this kind of drift, and (iii) leaving the row stale would have
  defeated the "audit pass that locks this in" framing of 23.2.
  Surfaced here so a future planner can re-validate.

- **Phase 17.5 (Code-Type Steering) is the last open 17.x sub-task
  and it's marked ❌ Missing — Phase 17 cannot seal until it
  lands or is explicitly deferred.** No new risk introduced by
