# Phase 19.5 Session Log: Strategy-Combination A/B Backtest Harness

- **Date**: 2026-05-03
- **Phase**: 19.5
- **Status**: ✅ Complete

## Scope

Phase 19.5 adds the offline payoff for the Phase 19 sub-account work:
run multiple sub-accounts over the same OHLCV window, preserve isolated
balances and ledgers, and emit a comparative report for strategy
combination A/B testing.

## Changes

- Added `src/backtest/harness.py` with `BacktestHarness.run_sub_accounts`.
  It filters strategies per `SubAccount.strategy_filter`, honors
  `initial_balance`, `risk_overrides.risk_percent`, and
  `risk_overrides.leverage_cap`, then returns a `MultiAccountReport`
  with per-sub-account metrics, equity curves, merged trade ledger,
  pairwise return correlations, and optional robustness verdicts.
- Added `src/backtest/multi_account_report.py` and exported
  `BacktestHarness` / `MultiAccountReport` from `src.backtest`.
- Extended `BacktestTrade` with default `sub_account_id` and
  `technique_name` attribution so merged combination ledgers can be
  traced back to both the capital pool and strategy.
- Added `scripts/backtest_combinations.py` operator entry. It loads
  the Phase 19.3 YAML sub-account shape, fetches one OHLCV window,
  runs the harness, and writes `report.json`, `trades.csv`, and a
  dependency-free `equity_curves.png`.
- Updated `src/dashboard/pages/strategies.py` to link the latest
  combination run and render side-by-side equity curves.
- Updated `docs/development-plan.md` and added the Phase 19 cross-check.

## Notes

The harness reuses the existing single-strategy `Backtester` per
sub-account strategy and combines each sub-account's results into an
isolated report surface. That keeps Phase 19.5 thin and low-risk while
still giving operators the comparative A/B artifact set required by
FR-038.
