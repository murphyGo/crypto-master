## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Separate TradeHistory model | Full trade lifecycle needs more fields than PerformanceRecord |
| Bidirectional linking | PerformanceRecord.trade_id ↔ TradeHistory.performance_record_id |
| Mode-based storage directories | Implements NFR-008 (backtest/paper/live separation) |
| JSON format for trades | Consistent with existing storage patterns |
| Default leverage=1 and fees=0 | Backward compatible with existing records |

## Implementation Details

### Storage Structure
```
data/trades/
    backtest/
        trades.json
    paper/
        trades.json
    live/
        trades.json
```

### TradeHistory Fields
- Entry: price, quantity, time, order_id
- Exit: price, quantity, time, order_id
- Financial: leverage, fees, pnl, pnl_percent
- Status: open/closed/cancelled, close_reason

### P&L Calculation (TradeHistory)
- Long: `(exit - entry) * qty * leverage - fees`
- Short: `(entry - exit) * qty * leverage - fees`
- Percentage: `((exit - entry) / entry) * 100 * leverage`
