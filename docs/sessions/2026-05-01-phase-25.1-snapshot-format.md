## Format Decision: CSV + JSON sidecar

Considered CSV / Parquet / JSONL. Picked CSV because:
- Snapshot files MUST be committed (the whole point is reproducibility),
  so version-control diff-friendliness is the dominant axis. Parquet is
  binary (PR review can't inspect row drift) and adds `pyarrow` as a dep.
- JSONL is ~2× verbose vs CSV with no schema-flexibility benefit since
  the row schema is fully fixed.
- CSV is human-readable, line-level diffable, ~2160 rows for a 90-day
  1h baseline (well within tractable PR-review size), and pandas (already
  a dep) handles read/write trivially.

Documented in module docstring + `data/backtest/snapshots/README.md`.

## Schema

**`ohlcv.csv`** header (pinned order): `timestamp,open,high,low,close,volume`.
- Timestamps as ISO-8601 UTC strings (Phase 21 contract).
- Price/volume cells as decimal strings (preserves `Decimal` round-trip;
  `float()` would drift).

**`metadata.json`** fields:
- `symbol` (canonical `BTC/USDT`)
- `timeframe` (Literal `15m | 1h | 4h`)
- `source`
- `fetched_at` (UTC-aware)
- `candle_count`
- `first_timestamp` / `last_timestamp` (UTC-aware)
- `fetcher_version` (`"phase-25.1"`)

Validation cross-checks: header order pinned; row column count enforced;
`metadata.candle_count` must equal actual CSV row count; UTC-coerce on
read via Pydantic `field_validator(mode="after")` (Phase 21.2 pattern).

## Spec Ambiguities Resolved

1. **Pydantic `ValidationError` wrapping**: spec said
   `SnapshotValidationError` on schema breach but pydantic raises
   `ValidationError`. Resolved: `_read_metadata` catches and re-raises
   as `SnapshotValidationError` so callers have a single exception type;
   pydantic message preserved in `__cause__`.
2. **Freshness boundary semantics**: spec didn't specify whether
   `age == max_age_days` is fresh. Chose `<=` (boundary fresh) and
   pinned in `test_is_snapshot_fresh_at_exact_boundary_is_fresh`.
3. **`baseline_directory` helper**: not explicitly in spec; added as
   single source of truth for the `<SYMBOL>__<timeframe>` convention so
   25.2's CLI flag has one place to call.
4. **gitignore mechanics**: spec said "add an exception" but
   directory-style `data/` exclusion blocked re-include. Switched to
   glob `data/*` + carve-backs.
