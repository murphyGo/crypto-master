## Work Summary

Added fee simulation to the local paper trading path so P&L reflects
realistic trading costs when no exchange testnet is available. The
PaperTrader now accepts a `FeeConfig` (maker/taker rates) and deducts
taker fees on position entry and exit. Per-exchange defaults are
pre-defined for Binance and Bybit; traders constructed without an
exchange or explicit config fall back to zero fees, preserving the
behavior of the earlier local-simulation tests.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Default to zero fees when no exchange is supplied | Preserves the existing local-simulation test contract; opt-in behavior keeps the change additive. |
| Look up per-exchange defaults via `exchange.name.lower()` | Matches the existing `BinanceExchange.name` / `BybitExchange.name` casing conventions and avoids a new registry abstraction. |
| Treat all market orders as taker fills | Paper trading only generates market orders today; maker path exists in `_calculate_fee` for future limit-order simulation. |
| Deduct entry fee from free balance (not margin) | Mirrors real exchange behavior where fees are charged on notional, separately from posted margin. |
| Rollback margin lock on fee deduction failure | Keeps balance state consistent if the trader is exactly at the margin threshold but cannot afford fees. |
| Do not modify `open_position_on_testnet` / `close_position_on_testnet` | Testnet exchange deducts real fees; simulating again would double-count. Real fee capture from order responses can come later. |

## Potential Risks

- Testnet path still passes `fees=0` to `close_trade`. When testnet fee
  data becomes available from order responses, it should be plumbed
  through to keep testnet and local P&L consistent.
- `close_position` zeroes out free balance when fees exceed available
  funds. This is consistent with the pre-existing behavior for losses
  exceeding margin but may mask edge cases in analytics.
