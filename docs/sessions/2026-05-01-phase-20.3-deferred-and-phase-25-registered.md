# Session Log: 2026-05-01 - Phase 20.3 - Deferred & Phase 25 Registered

## Work Summary

Phase 20.3 was originally scheduled as the closure for DEBT-029
("Phase 5.4+ baseline figures need re-computation post-leverage
fix") — a mechanical re-run of `scripts/backtest_baselines.py`
across the four documented baselines so that the inflated
absolute-PnL / equity / MDD-in-USDT figures (5–10× too large
under DEBT-024's leverage double-application) would be restated
post-fix. The senior-developer's pre-cycle inspection surfaced
three blockers that invalidated the "mechanical re-run" framing
entirely. Rather than ship a non-deterministic, operator-only
re-run that would break the autonomous-cycle contract and
produce drift-prone artefacts, the lead reframed the sub-task:
20.3 deferred, DEBT-029 closed as **Reframed** (not
implemented), the underlying reproducibility debt registered as
new DEBT-043 (Medium), and the actual design work split out
into Phase 25 (Snapshot-Pinned Reproducible Baselines). No
code changed this cycle; the work is entirely in the docs / debt
audit trail.

The three blockers were: (1) `scripts/backtest_baselines.py`
calls live Binance mainnet with no snapshot mode — the module
docstring at lines 26-30 states this explicitly, and the
live-exchange construction sits at lines 511-518. A re-run today
would produce non-deterministic output that drifts day-to-day
with whatever the live OHLCV looks like, which is not suitable
for an autonomous cycle and not idempotent across operators.
(2) No pre-DEBT-024 baseline JSONs exist on disk:
`data/backtest/baselines/` is gitignored and absent on this
checkout, and `docs/baselines.md`'s operator table at lines
124-136 is `_TBD_` for every metric. There is no inflated-figure
artefact to restate, so DEBT-029's "operator-facing artefact
regeneration" framing has zero current operator impact — the
bug existed in the math, not in any persisted operator surface.
(3) The Phase 20.3 spec listed 4 baselines but the script's
`BASELINES` table at lines 108-149 ships 5 (`rsi_universal`
extra), so even the "re-run the documented set" framing has a
silent spec-vs-code drift the cycle would have papered over.

The reframing splits the work cleanly. DEBT-029 closes as
**Reframed** because the original problem statement was wrong
in two ways: the math side was already closed at the code level
by Phase 20.1 + 20.2 (the `pnl_for_trade` helper, four PnL
sites routed through it, grep audit, convention docstrings,
regression-guard test — all verified by `TestPnLConventionAlignment`
numeric parity + the persistence-layer parity tests), and the
operator-artefact side was vacuous because no artefacts had
ever been persisted. The actual remaining debt is reproducibility:
the regenerator script needs to be deterministic across
operators / days before any baseline can be published as an
operator-facing reference. That is registered as DEBT-043
(Medium) and owned by Phase 25, which ships in three sub-tasks:
25.1 snapshot dataset + format (CSV / parquet / JSONL pick,
gitignore exception, freshness policy, format-validation test);
25.2 CLI `--snapshot` flag + script changes (snapshot loader
path, operator-gated `--refresh-snapshot` flag for the only
mainnet path, cross-operator determinism test, spec-vs-script
baseline-list reconciliation for the `rsi_universal` drift);
25.3 first run + populate `docs/baselines.md` (operator table
replaces every `_TBD_` with snapshot-pinned figure, change-
history row notes the first-time-post-DEBT-024-fix computation).

The cycle therefore seals Phase 20: 20.1 ✅ (math), 20.2 ✅
(discipline), 20.3 ⏸ Deferred (Phase 25). DEBT-024 is fully
closed at the code level — both math and discipline. The
trading-math correctness sweep is complete; the reproducibility
follow-up is correctly scoped as a separate phase rather than
absorbed into a sub-task that wasn't designed for it.

## Key Decisions

| Decision | Rationale |
|---|---|
| Defer Phase 20.3 rather than ship a live-Binance re-run | The "mechanical re-run" framing was wrong: non-deterministic output drifts day-to-day, breaks autonomous-cycle idempotence, and produces operator artefacts that are themselves a moving target. Better to flag the real debt (reproducibility) than to ship a one-off run that future cycles can't reproduce. |
| Close DEBT-029 as **Reframed**, not Resolved | The original problem statement (operator-facing baseline figures need restating) was vacuous — no figures had ever been persisted. Math side was already closed by 20.1 + 20.2. Marking it Resolved would imply implementation; marking it Reframed records the audit-trail truth that the framing itself was wrong. |
| Split reproducibility out as Phase 25, not absorb into 20.3 | Reproducible baselines need a snapshot format, gitignore exception, freshness policy, `--snapshot` CLI flag, and cross-operator determinism test — that's three sub-tasks of design work, not a "mechanical re-run". Keeping it as 20.3 would have either under-scoped the work or polluted Phase 20's "trading-math correctness" theme. |
| Mark Phase 20 ✅ Complete with footnote | 20.1 + 20.2 closed DEBT-024 fully at the code level; 20.3 is the right scope to defer because the math sweep is done. The footnote points future readers at Phase 25 for the reproducibility follow-up. Leaving Phase 20 ❌ Missing because of a deferred sub-task would mis-state the math work's status. |
| Reconcile `rsi_universal` spec drift inside Phase 25, not now | The spec-vs-script drift (4 documented, 5 in `BASELINES`) is a Phase 25.2 line item rather than a separate sub-task because it's tightly coupled to the snapshot-pinned regenerator's behaviour (which baselines does the snapshot cover?). Surface it now, fix it then. |
