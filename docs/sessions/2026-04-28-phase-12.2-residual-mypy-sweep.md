## Work Summary

Phase 12.2 closes the four follow-up clusters Phase 11.1 logged as
DEBT-005..008. Phase 11.1 cleared all in-scope ruff and mypy errors
(18 ruff → 0; 12 in-scope mypy → 0; total mypy 39 → 29 with
remainder out-of-scope per spec) but deliberately left four clusters
parked because each needed targeted handling rather than a
boilerplate sweep — DEBT-006 in particular was suspected as genuine
API-shape drift rather than typing hygiene and was kept on hold for
quant review. With the type-clean baseline established and no other
trading risk competing for the cycle, this is the right time to
clear the residual to zero. `mypy src` 29 errors → 0; ruff clean;
1109 tests pass unchanged (no behaviour change, no new tests — this
is a refactor, not a feature).

DEBT-005 (11 errors in `src/exchange/binance.py`) was the largest
cluster. The dev opted for the cleaner Protocol path over per-call
`# type: ignore[attr-defined]`: a hand-rolled `CCXTClient` Protocol
that structurally covers the 10 ccxt methods actually used by
`BinanceExchange` — `load_markets`, `close`, `fetch_ohlcv`,
`fetch_ticker`, `fetch_balance`, `create_market_order`,
`create_limit_order`, `cancel_order`, `fetch_order`,
`fetch_open_orders`. `_client` is now typed `CCXTClient | None`,
which lets mypy resolve attribute access without ccxt stubs. QA
verified all 10 call sites are covered by the Protocol.

DEBT-006 (3 errors in `src/exchange/factory.py`) was the
quant-review-needed cluster. The investigation outcome verbatim from
the dev: not a behavioural mismatch — the registry's
`type[BaseExchange]` widens away the subclass-specific `__init__`
parameters, so mypy correctly flags that the factory is calling
`exchange_class(...)` with kwargs the base class doesn't accept,
even though every concrete subclass does. Resolved with a
tightly-scoped `cast(Any, exchange_class)(...)` plus a comment
explaining the typing gap. Runtime preserves exact call shape; this
is a typing-system limitation rather than a real API drift, which
is why the quant-flagged investigation was the right call before
fix landed.

DEBT-007 (~13 errors clustered across
`src/dashboard/{theme,app,pages/trading,pages/engine}.py`) got the
straightforward annotations-and-casts treatment: `Literal` types
for theme constants (verified against streamlit's
`commands/page_config.py` signatures), `StreamlitPage` import for
the navigation chassis, `cast(...)` on `st.metric` numeric values
where mypy couldn't infer the narrowed type from
streamlit/pandas-derived values. DEBT-008 (`src/main.py:271` lambda
annotation, 1 error) got a targeted `# type: ignore[misc]` —
verbatim from QA, this is the canonical case for the asyncio
signal-handler callback shape mismatch and the right tool. Single
suppression; documented inline.

QA flagged one nice-to-have follow-up not in the original DEBT
list: the dashboard's `build_summary_metrics` returns
`dict[str, object]` and consumers therefore need
`cast(int, ...)` / `cast(float, ...)` at each access site. A
`TypedDict` rewrite would drop those casts. Recorded as DEBT-011
(Low) — pure type-system polish, no behaviour impact.

## Potential Risks

- **Protocol drift if ccxt adds/changes a method we use** — the
  hand-rolled `CCXTClient` Protocol covers exactly the 10 ccxt
  methods used today. If a future cycle reaches for an 11th method
  on `_client` it will fail mypy until the Protocol is extended,
  which is the desired behaviour: every new ccxt surface is
  consciously added rather than silently picked up. Worth keeping
  on the radar when extending `BinanceExchange`.
- **Factory `cast(Any, ...)` masks future real drift** — the cast is
  tightly scoped and commented, but it does mean a genuine API
  mismatch (e.g. a subclass `__init__` losing a kwarg the registry
  caller still supplies) would no longer fail mypy. Acceptable
  given the typing gap is a real limitation; mitigation is
  test-suite coverage of factory construction paths (already in
  place).
- **DEBT-011 follow-up** — `build_summary_metrics` returning
  `dict[str, object]` forces consumers to `cast(int, ...)` /
  `cast(float, ...)`. Each cast is a tiny lie the type system
  relies on; a `TypedDict` rewrite would drop all of them. No
  runtime impact today.
