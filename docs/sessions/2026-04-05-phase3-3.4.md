## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Store records per technique in subdirectories | Organize data and allow easy deletion per technique |
| Use JSON for storage | Per NFR-006 requirement, human-readable |
| Calculate P&L as percentage | Normalizes across different price levels |
| Separate records.json and summary.json | Records are append-only, summary is recalculated |
| Use UUID for record IDs | Unique identification without database |

## Implementation Details

### Storage Structure
```
data/performance/
    {technique_name}/
        records.json     # List of PerformanceRecord
        summary.json     # TechniquePerformance aggregate
```

### Performance Metrics
- `win_rate`: wins / (wins + losses + breakevens)
- `avg_pnl_percent`: average P&L percentage per closed trade
- `total_pnl_percent`: cumulative P&L percentage
- `best_trade_pnl` / `worst_trade_pnl`: extremes

### P&L Calculation
- Long trades: ((exit - entry) / entry) * 100
- Short trades: ((entry - exit) / entry) * 100
