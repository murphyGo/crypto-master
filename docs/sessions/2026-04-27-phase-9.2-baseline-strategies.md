## Work Summary

Added three deterministic indicator strategies as comparison
baselines for the LLM-driven techniques:

- **`rsi_mean_reversion`** ‚Äî Wilder's RSI(14), long < 30, short > 70.
- **`bollinger_band_reversion`** ‚Äî Bollinger Bands (20, 2œÉ),
  long below lower band, short above upper band, target the SMA.
- **`ma_crossover`** ‚Äî SMA(10) / SMA(20) golden / death cross.

All three sit under `strategies/`, marked `status: experimental`
and `symbols: []` (universal), so the engine applies them on every
symbol scanned. Math lives in a shared `src/strategy/indicators.py`
module (`rsi`, `sma`, `bollinger_bands`) so the strategies don't
duplicate arithmetic.

The original `sample_code.py` was promoted into the proper
`ma_crossover.py` baseline ‚Äî same logic, cleaner framing, integration
test path updated.

`docs/baselines.md` documents the three, includes a `Backtester`
invocation snippet for populating win-rate / Sharpe / MDD numbers,
and reserves a results table to fill in once an operator runs the
actual backtests (historical-OHLCV fetching is a separate small
task ‚Äî flagged as a follow-up).

## Scope adjustment

The original Phase 9.2 plan listed four strategies (`rsi_4h`,
`rsi_15m`, `bollinger_bands`, `ma_crossover`). Without Phase 9.1
multi-TF support the engine fetches one timeframe per cycle, so a
`rsi_4h` and a `rsi_15m` would in practice analyse the same data ‚Äî
the metadata would lie. Collapsed to a single `rsi.py` for now,
with the per-timeframe split documented as a Phase 9.1 follow-up.
Plan + history rows updated accordingly.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Shared `indicators.py` (not per-strategy math) | Three strategies use SMA-derived math; duplicating Wilder's RSI in two files is asking for a divergence bug. Closed-form, dependency-free, fast import ‚Äî same shape as `src.feedback.audit` style helpers. |
| `population` stdev for Bollinger Bands | Matches TradingView, ta-lib, and most Python TA libs. Sample stdev would give slightly wider bands and surprise anyone double-checking against a chart. |
| Wilder's RSI smoothing (not the SMA variant) | Same canonical reasoning ‚Äî TradingView and ta-lib default to Wilder's. Using the SMA variant would produce different numbers from what an operator sees on their charts. |
| Single `rsi.py` instead of `rsi_4h.py` + `rsi_15m.py` | Without Phase 9.1 multi-TF support, both files would analyse the same engine-configured timeframe. Better to have one honest strategy now and split it cleanly post-9.1. |
| Promoted `sample_code.py` ‚Üí `ma_crossover.py` (rename + delete) | The "sample_code" name signalled scaffolding; once it's a real registered baseline the framing matters. Rename + integration-test path update is a 5-line change. |
| Fixed-percent SL/TP on RSI (2% / 4%) | Constant R/R across symbols/timeframes. The baseline's whole job is being a clean comparison floor ‚Äî variable SL/TP would muddle "is the LLM better at *timing* or just at *risk sizing*?". |
| Band-relative SL/TP on Bollinger | Mean-reversion's natural shape: TP is the mean, SL sits half a band outside the trigger so a continuation invalidates the thesis. |
| Backtest numbers deferred (not bundled) | Running the Backtester needs historical OHLCV which we don't have a fetcher for yet. Bundling that into 9.2 would balloon the scope. `docs/baselines.md` shows the exact invocation an operator runs, results go in `data/backtest/baselines/`. |
| `_neutral_result` helper inside each strategy file | Pydantic's `AnalysisResult` requires entry/SL/TP > 0 even on neutral signals. Abstracting "give me a valid neutral result" into a tiny helper avoids three near-identical inline blocks per strategy. |

## Potential Risks

1. **Per-strategy R/R defaults are fixed at 2:1.** Real markets
   may want symbol-specific tuning. Acceptable starting point;
   revisit once we have backtest numbers to compare across pairs.
2. **`min_candles` thresholds are conservative**: RSI requires
   `period * 3 = 42` candles, BB requires 20, MA requires 21. A
   fresh symbol with only a few hours of history will return
   neutral (informational, not a crash).
3. **Confidence formulas are heuristic.** Linear ramps with
   arbitrary endpoints. Once the engine has real proposal-vs-outcome
   data, calibrate from there.
4. **No backtest results bundled.** `docs/baselines.md` reserves
   the comparison table but the actual numbers depend on the
   operator running the documented `Backtester` invocation. A
   thin "fetch + backtest + write" script is the natural follow-up
   sub-task.
5. **chasulang remains dormant** ‚Äî 9.1 is the path to revive it.
   The three baselines are independent of chasulang's status.
