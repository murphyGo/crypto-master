## Work Summary

Enhanced PaperTrader to optionally use exchange testnets for order execution instead of pure local simulation. This enables more realistic paper trading by executing orders against actual testnet APIs.

- Added `sync_balance_from_exchange()` async method:

- Added `open_position_on_testnet()` async method:

  - Maps position side to order side (long→buy, short→sell)
  - Records trade with real exchange order ID
  - Handles insufficient funds error specially
- Added `close_position_on_testnet()` async method:
  - Creates closing market order (opposite side)
  - Updates trade history with exit price
  - Removes position from tracking

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Optional exchange parameter | Maintains backward compatibility with existing code |
| Async methods for testnet ops | Exchange operations are inherently async |
| Keep local methods synchronous | Existing local simulation doesn't need async overhead |
| TYPE_CHECKING import | Avoids circular import between paper.py and exchange/base.py |
| Market orders only | Simpler implementation; limit orders can be added later |
| Store exchange order ID | Enables order status tracking if needed in future |
| Sync balance on-demand | Avoids unexpected API calls; explicit is better |

## Implementation Details

### Order Execution Flow (Testnet Mode)

```
open_position_on_testnet(position)
  │
  ├─ Verify testnet mode enabled
  ├─ Extract quote currency from symbol
  ├─ Build OrderRequest(side=long→buy/short→sell, type=market)
  ├─ Call exchange.create_order()
  ├─ Calculate margin for internal tracking
  ├─ Record trade via TradeHistoryTracker with real order ID
  ├─ Track open position
  └─ Return TradeHistory
```

### Balance Sync Flow

```
sync_balance_from_exchange(currency?)
  │
  ├─ Verify testnet mode enabled
  ├─ Call exchange.get_balance(currency)
  ├─ For each Balance returned:
  │     └─ Update _balances[currency] = PaperBalance(free=..., locked=...)
  └─ Log sync completion
```
