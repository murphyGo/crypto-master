## Restructured `docs/baselines.md`

New sections added:
- **Operator runbook — first-time fetch + run** (5-step procedure):
  1. Set `BINANCE_API_KEY` / `BINANCE_API_SECRET` (read-only OK).
  2. `python -m scripts.backtest_baselines --refresh-snapshot
     --snapshot-root data/backtest/snapshots/` (sole mainnet path).
  3. Verify `BTCUSDT__{15m,1h,4h}/` subdirectories — 3 not 5,
     because `rsi_universal`, `bollinger_band_reversion`, and
     `ma_crossover` share the `BTC/USDT 1h` snapshot.
  4. `python -m scripts.backtest_baselines --snapshot
     data/backtest/snapshots/` to compute and persist artefacts;
     rewrites table in place.
  5. `git add data/backtest/snapshots/baselines/
     data/backtest/baselines/ docs/baselines.md && git commit`.
- **Snapshot freshness policy**: 30-day active-use window
  (`--max-snapshot-age-days` default; quant-recommended for
  promotion-gate baselines) vs 90-day absolute stale ceiling
  (`DEFAULT_MAX_AGE_DAYS` in `src/backtest/snapshot.py`). Operators
  refresh on a 30-day cadence; snapshots older than 90 days will be
  refused by the loader regardless.
- **Reproducibility note**: two operators running against the same
  committed snapshot produce byte-identical metrics + ledger; UUIDs
  (`run_id`, `trade_id`) differ but trade-level numbers are
  byte-identical (matches the `test_cross_operator_determinism_byte_
  identical` contract from 25.2).
