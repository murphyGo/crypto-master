## Work Summary

Built the `LiveTrader` class that executes real orders against a
mainnet exchange. Every user-initiated action (open, manual close)
funnels through an async confirmation callback before any order is
sent, satisfying NFR-012. A `monitor_positions()` method polls the
exchange ticker for each open position and auto-closes on SL/TP
without re-prompting (those bounds were pre-approved at open time).
All trades are recorded via the shared `TradeHistoryTracker` under
`mode="live"`, giving the existing mode-separated history (NFR-008) a
live counterpart with no tracker changes.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| `confirmation_callback` is an injected async callable, default = CLI prompt | Keeps NFR-012 explicit and testable; callers on CI/dashboard can provide their own UI. |
| Reject testnet exchanges in `__init__` with `LiveModeError` | Prevents accidentally running "live" code against sandbox endpoints. Testnet belongs to `PaperTrader`. |
| Confirmation required for open + manual close, NOT for SL/TP auto-exit | Mirrors real exchange behavior — SL/TP are pre-authorized protective stops. Re-prompting would delay risk exits. |
| Track `Position` objects in-memory keyed by trade_id (not persisted) | `TradeHistory` stores the closed-loop facts; the live SL/TP monitor only needs the active bounds, which live in the session. If the process restarts, the user re-attaches explicitly — a safer default than silently replaying stored bounds. |
| `_submit_order` wraps exchange exceptions into `LiveOrderRejectedError` and also checks status for `REJECTED` / `CANCELLED` | Gives callers a single exception type for "exchange didn't accept this" independent of how the failure surfaced. |
| `monitor_positions` swallows per-position failures and logs them | One bad symbol shouldn't break the whole monitoring pass. |
| Use `TradeHistoryTracker` with `mode="live"` (no new storage layer) | NFR-008 already enforces mode separation at the tracker; live trades slot into `data/trades/live/` with no changes. |
| Default CLI prompt uses `asyncio.to_thread(input, …)` | Prevents the sync `input()` from blocking the event loop while still being simple to use. |

## Potential Risks

- **Fee capture**: `LiveTrader` currently passes no `fees` value to
  `close_trade`. Real exchange fees are charged outside our accounting
  for now. Tracking real fees requires adding a `fees` field to the
  `Order` model and reading it back from `create_order` responses.
- **Market-order price recording**: `entry_price` / `exit_price` on the
  recorded trade use the requested price, not the actual fill price.
  `Order` does not yet carry an average-fill field; once the exchange
  integrations expose it, both LiveTrader and the testnet path should
  switch to it for accurate P&L.
- **In-memory position state is not persisted**: If the process
  restarts, open live positions lose their tracked SL/TP until the
  user re-attaches them. This is a conscious choice — blindly
  reloading bounds from disk on restart could reopen automated trades
  the user did not explicitly acknowledge.
- **No balance precheck before open**: We rely on the exchange to
  reject under-funded orders. Adding a proactive `get_balance` check
  would improve UX but adds an extra API call per open.

## TECH-DEBT Items

No new debt items added. The risks above are documented here and can
