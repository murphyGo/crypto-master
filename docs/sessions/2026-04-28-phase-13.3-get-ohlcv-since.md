# Session Log: 2026-04-28 - Phase 13.3 - BaseExchange.get_ohlcv `since` Parameter (DEBT-004)

## Overview

- **Date**: 2026-04-28

## Work Summary

Phase 13.3 closes DEBT-004 by extending the framework's OHLCV fetch
contract so `scripts/backtest_baselines.py` can drop the `_client`
reach-around it carried since Phase 10.3. The ABC now declares
`since: int | None = None` on `BaseExchange.get_ohlcv`; both
production adapters (`BinanceExchange`, `BybitExchange`) forward the
parameter to `ccxt.fetch_ohlcv(since=...)`. Default behaviour is
preserved bytewise — existing callers that omit `since` get the
same most-recent-page semantics they had pre-13.3, locked by the
`test_get_ohlcv_defaults_since_to_none` tests for both adapters.

The script's pagination loop previously dropped down to
`exchange._client.fetch_ohlcv(...)` to access ccxt's `since` cursor
directly because the abstraction didn't expose it. With the
abstraction now sufficient, the loop calls `exchange.get_ohlcv(...,
since=...)` end-to-end. The script no longer constructs `OHLCV`
objects from raw ccxt rows — `BinanceExchange.get_ohlcv` already
does the conversion — so the local `Decimal` import and the
`datetime`/`Decimal`-based reconstructor at the bottom of
`fetch_ohlcv_window` are gone. The `RuntimeError` raised when
`_client` was missing is also gone (no longer reachable). The
docstring and comments updated to reflect the public-API path; the
"deliberately reach past the BaseExchange contract" comment is
deleted.

Test coverage adds 4 new tests — 2 per adapter — covering both
default-`None` forwarding and explicit-`since` forwarding via
`mock_ccxt_client.fetch_ohlcv.call_args[1]` assertions. The
`MockExchange` in `tests/test_exchange_base.py` and the
`_FakeBinanceExchange` in `tests/test_scripts_backtest_baselines.py`
both grew the new `since` parameter to satisfy the ABC; the latter
also absorbed the pagination-cursor logic that previously lived in
the deleted `_FakeCCXTClient`. The smoke test for the script's
multi-page path stays green without modification — the public-API
shape it asserts on (final flattened ascending OHLCV list) is
unchanged.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Default `since=None` matches pre-13.3 behaviour bytewise | Existing callers (engine, dashboard, strategy code) all omit `since` today; the change must be a pure addition. Locked by `test_get_ohlcv_defaults_since_to_none` for both Binance and Bybit. |
| Update Bybit at the same time as Binance even though only Binance has a current consumer | The ABC change forces *some* answer in `BybitExchange.get_ohlcv`. Forwarding to ccxt costs nothing, keeps the two adapters symmetric, and prevents a "Bybit ignores since" footgun if a future consumer paginates through Bybit. Tests cover both. |
| Drop the script's raw-row → `OHLCV` reconstructor entirely | `BinanceExchange.get_ohlcv` already returns `OHLCV` objects; once the script no longer touches `_client`, both pagination branches return `OHLCV` lists directly, so the bottom-of-function dataclass-rebuilder + the `Decimal` import are dead code. |
| Keep `RuntimeError` deletion silent (no replacement) | The error gated the `_client is None` case, which only happened when `connect()` hadn't been called. The new path goes through `exchange.get_ohlcv`, which has its own connection check raising the appropriate adapter-specific error. No coverage gap. |
| Paginate via timestamp arithmetic on `OHLCV.timestamp` rather than raw ms | Switching the script away from raw ccxt rows means `earliest_ts` now derives from `recent[0].timestamp.timestamp() * 1000` (round-tripped through int). The semantics match the previous `raw_recent[0][0]` exactly — both give the millisecond timestamp of the earliest candle in the page. |
| Smoke test for script's pagination unchanged | The test's contract is "feed the script N candles, get an ascending de-duped list back". The internal switch from `_client` to `get_ohlcv` doesn't change the contract; only the fake exchange's surface changes (drop `_client`, add `since` param). |

## Potential Risks

- **ccxt `since` semantics drift between adapters** — both Binance
  and Bybit ccxt classes accept `since` as inclusive-on-start ms,
  matching what the docstring promises. If a future ccxt upgrade
  changes the semantics on one but not the other, the abstraction
  papers over a real difference. The forwarding tests pin the call
  shape (`since` value reaches ccxt verbatim) but not the
  exchange-side interpretation. Acceptable; ccxt is the canonical
  abstraction here, not us.
- **Bybit's 200-candle cap is much smaller than Binance's 1500** —
  callers that paginate Bybit will need many more pages for the
  same window than callers that paginate Binance. Phase 13.3 just
  lights up the parameter; if a future consumer relies on Bybit
  pagination at length, they should add a similar pagination
  helper. Out of scope here.
- **Script's smoke test no longer covers the
  raw-row → `OHLCV` conversion path** — that path is gone, so
  there's nothing left to cover; `BinanceExchange.get_ohlcv` owns
  the conversion and has its own coverage. Not a real risk; called
  out for the audit.
