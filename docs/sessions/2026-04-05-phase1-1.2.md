## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Use Pydantic Settings | Type-safe configuration, automatic validation, .env file support (NFR-004) |
| Nested exchange configs | Clean separation of credentials per exchange with dedicated prefixes |
| Singleton pattern for settings | Single source of truth, avoid repeated file reads |
| `is_configured()` method | Easy check if API credentials are set without exposing values |
| `validate_for_live_trading()` | Explicit validation before enabling live mode (NFR-012) |
| Empty string defaults for API keys | Allow paper trading without any exchange configuration |
