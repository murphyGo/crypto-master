## Goal

Land the architectural seam for Phase 19's multi-sub-account fan-out
without touching engine, persistence, or dashboard behaviour. The
deliverable is: a `SubAccount` Pydantic entity, a
`SubAccountRegistry` that today materialises a single `default`
sub-account from `Settings`, and a one-shot idempotent on-disk
migration that renames legacy `data/{trades,portfolio,proposals}/
{mode}/...` paths into `.../default/...` guarded by a marker file.

The single-default registry preserves the legacy single-seed
back-compat invariant (one `PaperTrader` instance, addressed by
`id="default"`); multi-account parsing is 19.3 territory, engine
fan-out is 19.2.

## Implementation

### `SubAccount` entity (`src/trading/sub_account.py`, new)

- `SubAccount` Pydantic model with fields per `DESIGN.md §9.2`:
  `id` / `name` / `mode` / `exchange_ref` / `initial_balance` /
  `strategy_filter` / `risk_overrides` / `enabled`.
- `RiskOverrides` sub-model with optional `risk_percent` /
  `max_open_positions_total` / `max_open_positions_per_symbol` /
  `leverage_cap`.
- Validators: `id` matches `^[a-z][a-z0-9_]*$` (filesystem-safe);
  `enabled and mode == "live"` requires non-null `exchange_ref`;
  `initial_balance` keys are upper-case currency codes.
- Frozen model (`model_config = ConfigDict(frozen=True)`) so
  registry consumers can't mutate behind the registry's back.
- `SubAccountNotFoundError` exception type defined alongside.

### `SubAccountRegistry` (`src/trading/sub_account_registry.py`, new)

- `__init__(settings, trader, config_path: Path | None = None)`;
  default `config_path` is `Path("config/sub_accounts.yaml")`.
- When config file is absent (the 19.1 default — YAML support is
  19.3 territory), materialises one
  `SubAccount(id="default", name="Default Account",
  mode=settings.trading_mode, exchange_ref="default",
  initial_balance={"USDT": Decimal(str(settings.paper_initial_balance))},
  strategy_filter=None, risk_overrides=RiskOverrides(),
  enabled=True)`.
- Methods: `list_active()`, `get(id)` (raises
  `SubAccountNotFoundError`), `get_trader(id)` (returns the single
  shared `Trader` in 19.1 — registry stores the wiring seam now,
  19.2 lights it up), `filter_strategies(id, available)` (returns
  `available` unchanged when `strategy_filter is None`).
- `_load` carries a 19.3 placeholder branch
  (`if self.config_path.exists(): pass`). Inert in 19.1; documented
  in the docstring (recorded as TECH-DEBT — see below).

### `migrate_legacy_paths` (`src/trading/sub_account_migration.py`, new)

- `migrate_legacy_paths(data_dir: Path) -> dict[str, int]`.
- Idempotent rename of:
  - `data/trades/{mode}/trades.json` →
    `data/trades/{mode}/default/trades.json`
  - `data/portfolio/{mode}/snapshots.json` →
    `data/portfolio/{mode}/default/snapshots.json`
  - `data/proposals/{date}_{symbol}.json` →
    `data/proposals/default/{date}_{symbol}.json`
- Across `mode in {paper, live, backtest}`.
- Performance subtree (`performance/{technique}/...`) deferred to
  19.2 because the new layout adds a sub-account level above the
  technique.
- Marker file `data_dir/.subaccounts_migrated_v19_1` written on
  first success; presence short-circuits subsequent invocations.
- Returns count-per-component dict for operator log.

### Engine wiring (`src/main.py`, modified)

- `migrate_legacy_paths(settings.data_dir)` invoked before
  `build_engine` (mirrors Phase 11.4's `_purge_old_proposals`
  placement). INFO log on non-zero rename count only (silent on
  already-migrated to avoid restart noise).
- Registry constructed after `build_trader` returns; passed to
  `build_engine`. Registry currently unused inside the engine —
  the parameter is added now so 19.2 doesn't churn `build_engine`'s
  signature.
- `build_engine` restructured: gained optional `registry`,
  `trader`, `activity_log` kwargs (all back-compat — defaults
  preserve the prior wiring); extracted `_engine_config_from_settings`
  helper.
- **Engine attr hack**: `src/main.py:339` sets
  `engine.sub_account_registry = registry  # type: ignore[attr-defined]`
  because `TradingEngine` doesn't expose the attribute yet. Per
  the dev's report, 19.2 will lift this into a proper
  `TradingEngine.__init__` parameter (recorded as TECH-DEBT —
  see below).
