## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Keep ccxt's sandbox parameter | ccxt already handles URL routing internally |
| Add URL constants for documentation | Makes testnet URLs visible/discoverable |
| Separate testnet API keys | Testnet/mainnet may have different credentials |
| Fallback to live keys | If testnet keys not set, use live keys for backward compatibility |
| BaseExchange stores testnet flag | Standardizes testnet handling across all exchanges |

## Implementation Details

### Credential Selection Flow
```
config.get_credentials()
  ├─ If testnet=True AND testnet_api_key is set:
  │     Return (testnet_api_key, testnet_api_secret)
  └─ Else:
        Return (api_key, api_secret)
```

### Exchange Connect Flow
```
exchange.connect()
  ├─ api_key, api_secret = config.get_credentials()
  └─ ccxt_client = ccxt.exchange({
       "apiKey": api_key,
       "secret": api_secret,
       "sandbox": self.testnet,  # Routes to testnet URLs
       ...
     })
```
