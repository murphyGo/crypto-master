## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Use existing Settings for defaults | Config already has max_leverage, max_position_size_pct, default_stop_loss_pct |
| Risk-based position sizing | Standard formula: quantity = risk_amount / risk_per_unit |
| Cap position at max % of balance | Prevent over-exposure even with high leverage |
| Allow validation bypass | Useful for backtesting edge cases |
| Decimal for all monetary values | Financial precision |

## Implementation Details

### Position Sizing Formula
```
risk_amount = balance * (risk_percent / 100)
risk_per_unit = |entry_price - stop_loss|
quantity = risk_amount / risk_per_unit

# Margin validation
margin_required = (quantity * entry_price) / leverage
max_margin = balance * (max_position_size_percent / 100)
if margin_required > max_margin:
    quantity = (max_margin * leverage) / entry_price
```
