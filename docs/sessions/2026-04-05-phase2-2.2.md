## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Use ccxt library | Already a project dependency; provides unified API for multiple exchanges |
| Dual client support (spot/futures) | BinanceConfig.market_type determines client class (ccxt.binance vs ccxt.binanceusdm) |
| Enable ccxt rate limiting | Built-in rate limit handling via `enableRateLimit: True` satisfies CON-002 |
| Return client from _ensure_connected() | Type narrowing for mypy; ensures client is not None after check |
| Use sandbox mode for testnet | ccxt handles testnet URLs automatically when sandbox=True |

## Implementation Details

### BinanceExchange Features:
- **Connection Management**: Async connect/disconnect with context manager support
- **Market Data**: get_ohlcv (max 1500 candles), get_ticker
- **Account Data**: get_balance with optional currency filter
- **Order Management**: create_order (market/limit), cancel_order, get_order, get_open_orders
- **Error Handling**: Maps ccxt exceptions to project exception hierarchy

### Error Code Mapping:
| ccxt Exception | Project Exception | Code |
|----------------|-------------------|------|
| AuthenticationError | ExchangeConnectionError | - |
| NetworkError | ExchangeConnectionError | - |
| RateLimitExceeded | ExchangeAPIError | RATE_LIMIT |
| InvalidOrder | ExchangeAPIError | INVALID_ORDER |
| OrderNotFound | ExchangeAPIError | NOT_FOUND |
| InsufficientFunds | ExchangeAPIError | INSUFFICIENT_FUNDS |
